# Currency Training Data

Synthetic note patterns were removed because they produced misleading validation results and did not represent Indian banknotes. The local research dataset is now prepared from the publisher-labelled [Indian Currency Real vs Fake Notes Dataset](https://www.kaggle.com/datasets/preetrank/indian-currency-real-vs-fake-notes-dataset), licensed CC BY-NC-SA 4.0.

Run the reproducible ingestion command from the repository root:

```powershell
python backend/data/scripts/prepare_currency_dataset.py --per-stratum 43
```

The command selects a balanced subset across INR 10, 20, 50, 100, 500, and 2000 denominations, rejects unreadable/undersized/exact-duplicate files, and writes `source_manifest.json`. Images are intentionally Git-ignored and must be regenerated by each developer.

Installed layout:

```text
currency/
  genuine/
  counterfeit/
```

Do not place augmentations of the same physical note across train and validation splits. The training script uses perceptual-hash groups and `StratifiedGroupKFold` to reduce leakage between near-duplicate captures.

## Label assurance

These are publisher labels from a noncommercial research dataset, not RBI, bank, police, or forensic-laboratory certifications. Some counterfeit examples are movie/prop notes and are useful for screening demonstrations but do not represent every real counterfeit process. Reported validation metrics are research metrics only; production deployment requires an independently governed, bank/lab-verified holdout collected under representative devices and lighting.
